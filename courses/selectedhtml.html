What is HTML?

HTML (HyperText Markup Language) is the standard markup language for creating web pages and web applications. It describes the structure of web pages using markup and is the backbone of all websites.

History of HTML

HTML was created by Tim Berners-Lee in 1991 while working at CERN. Since then, it has evolved through several versions:

  • HTML 2.0 (1995): First standardized version, providing basic structure and form elements.
  • HTML 3.2 (1997): Introduced tables, applets, and text flow, enhancing layout capabilities.
  • HTML 4.01 (1999): Major revision with CSS support, allowing for better styling and layout.
  • XHTML (2000): XML-based version that enforced stricter syntax rules.
  • HTML5 (2014): Current standard with multimedia support, semantic elements, and new APIs for web applications.

Key Features of HTML

  • Markup Language: Uses tags to define elements, making it easy to structure content.
  • Platform Independent: Works across all devices and browsers, ensuring accessibility.
  • Easy to Learn: Simple syntax with clear structure, making it beginner-friendly.
  • Extensible: Can be combined with CSS for styling and JavaScript for interactivity.
  • Semantic: HTML5 introduced meaningful elements that describe the content's purpose.
  • Multimedia Support: Native audio and video elements allow for rich media experiences.

How HTML Works?

  1. Writing HTML: Developers write HTML code in text editors or IDEs, using tags to structure content.
  2. Browser Rendering: Web browsers parse HTML and render it visually, converting the markup into a user-friendly interface.
  3. User Interaction: Users interact with the rendered page through clicks, form submissions, and navigation.

Conclusion

HTML is the foundation of web development. Understanding HTML is essential for anyone looking to create websites or web applications.

HTML Basics

HTML documents are made up of elements that define the structure and content of a web page.

Basic HTML Document Structure

A basic HTML document consists of the following structure:

<!DOCTYPE html>  
<html>           
<head>           
    <title>Page Title</title>  
</head>
<body>           
    <h1>My First Heading</h1>  
    <p>My first paragraph.</p>  
</body>
</html>

Key Components

  • <!DOCTYPE html>: Declares the document type and HTML version, ensuring proper rendering.
  • <html>: The root element of an HTML page, containing all other elements.
  • <head>: Contains meta information about the document, such as the title and links to stylesheets.
  • <title>: Specifies a title for the document, displayed in the browser's title bar or tab.
  • <body>: Contains the visible page content, including text, images, and other elements.

HTML Tags

HTML tags are element names surrounded by angle brackets:

  • Tags normally come in pairs like <p> and </p> (start and end tags).
  • The first tag is the start tag, and the second is the end tag, which closes the element.
  • Some tags are self-closing, like <img> and <br>, meaning they do not require an end tag.

HTML Elements

HTML elements are the building blocks of HTML pages, represented by tags.

Element Structure

The basic structure of an HTML element is:

<tagname>Content goes here...</tagname>

Common HTML Elements

  • <h1> to <h6>: Headings of different levels, with <h1> being the highest and <h6> the lowest.
  • <p>: Paragraph element used for blocks of text.
  • <a>: Hyperlink element used to link to other pages or resources.
  • <img>: Image element used to embed images in a page.
  • <div>: Division element used as a container for other elements.
  • <span>: Inline container used to group inline elements.
  • <ul>, <ol>, <li>: Unordered and ordered lists, with list items.
  • <table>, <tr>, <td>: Table elements for displaying tabular data.
  • <form>, <input>: Form elements for user input.

Nested Elements

Elements can contain other elements, allowing for complex structures:

<div>  
    <h1>Title</h1>  
    <p>Paragraph with <strong>bold</strong> text.</p>  
</div>

HTML Attributes

Attributes provide additional information about HTML elements.

Attribute Structure

Attributes are specified in the opening tag of an element:

<tagname attribute="value">Content</tagname>

Common Attributes

  • id: Specifies a unique id for an element, used for styling and scripting.
  • class: Specifies one or more class names for styling with CSS.
  • style: Specifies inline CSS styles for an element.
  • src: Specifies the source for embedded content, such as images or scripts.
  • href: Specifies the URL for links, directing users to other pages.
  • alt: Alternative text for images, displayed if the image cannot be loaded.
  • width/height: Specifies dimensions for images and other elements.

Example

<a href="https://www.example.com" target="_blank">Visit Example</a>  
<img src="image.jpg" alt="Description" width="500" height="600">  
<p class="intro" id="first-paragraph">This is a paragraph.</p>

Headings & Paragraphs

Headings and paragraphs are fundamental for organizing text content.

Headings

HTML has six levels of headings, with <h1> being the most important and <h6> the least:

<h1>Main Heading</h1>  
<h2>Subheading</h2>  
<h3>Sub-subheading</h3>  
<h4>Level 4 Heading</h4>  
<h5>Level 5 Heading</h5>  
<h6>Level 6 Heading</h6>

Paragraphs

The <p> element defines a paragraph:

<p>This is a paragraph.</p>  
<p>This is another paragraph.</p>

Line Breaks

The <br> tag inserts a single line break:

<p>This is<br>a line<br>break example.</p>

Horizontal Rule

The <hr> tag defines a thematic break, often used to separate content:

<p>First section</p>
<hr>  
<p>Second section</p>

Styles & Formatting

HTML provides elements for styling and formatting text.

Text Formatting

  • <b>: Bold text, used for strong emphasis.
  • <strong>: Important text (semantic), typically displayed as bold.
  • <i>: Italic text, used for emphasis or titles.
  • <em>: Emphasized text (semantic), typically displayed as italic.
  • <mark>: Marked/highlighted text, used to highlight parts of the text.
  • <small>: Smaller text, often used for fine print.
  • <del>: Deleted text, shown with a strikethrough.
  • <ins>: Inserted text, shown with an underline.
  • <sub>: Subscript text, used for chemical formulas.
  • <sup>: Superscript text, used for exponents.

Example

<p><b>Bold</b> and <strong>strong</strong> text.</p>
<p><i>Italic</i> and <em>emphasized</em> text.</p>
<p>This is <mark>marked</mark> text.</p>
<p>This is <del>deleted</del> and <ins>inserted</ins> text.</p>
<p>H<sub>2</sub>O and x<sup>2</sup>.</p>

HTML Styles

The style attribute allows inline CSS styles to be applied directly to elements:

<p style="color:red;">This is a red paragraph.</p>  
<p style="font-family:courier;">This is courier font.</p>  
<p style="text-align:center;">Centered paragraph.</p>  
<p style="font-size:50px;">Big text.</p>

Links & Images

Links and images are essential for navigation and content in web pages.

Creating Links

Use the <a> tag to create hyperlinks:

<a href="https://www.example.com">Visit Example</a>

You can also use the target attribute to specify where to open the linked document:

<a href="https://www.example.com" target="_blank">Open in New Tab</a>

Images

Use the <img> tag to embed images in a page:

<img src="image.jpg" alt="Description" width="500" height="600">

The alt attribute provides alternative text for the image, which is important for accessibility.

Lists

HTML supports ordered and unordered lists for organizing content.

Unordered Lists

Use the <ul> tag to create an unordered list:

<ul>
    <li>Item 1</li>  
    <li>Item 2</li>  
    <li>Item 3</li>  
</ul>

Ordered Lists

Use the <ol> tag to create an ordered list:

<ol>
    <li>First item</li>  
    <li>Second item</li>  
    <li>Third item</li>  
</ol>

Tables

HTML tables are used to display data in a structured format.

Table Structure

A basic table consists of rows and cells:

<table>  
    <tr>  
        <th>Header 1</th>  
        <th>Header 2</th>  
    </tr>
    <tr>  
        <td>Data 1</td>  
        <td>Data 2</td>  
    </tr>
</table>

You can also use the colspan and rowspan attributes to merge cells:

<tr>
    <td colspan="2">Merged Cell</td>  
</tr>

Forms

HTML forms are used to collect user input.

Form Structure

A basic form consists of various input elements:

<form action="/submit" method="post">  
    <label for="name">Name:</label>  
    <input type="text" id="name" name="name" required>  
    <input type="submit" value="Submit">  
</form>

Forms can include various input types, such as:

  • <input type="text">: For single-line text input.
  • <input type="email">: For email input, with validation.
  • <input type="password">: For password input, obscured text.
  • <input type="radio">: For selecting one option from a set.
  • <input type="checkbox">: For selecting multiple options.
  • <select>: For dropdown lists.
  • <textarea>: For multi-line text input.

Semantic HTML

Semantic HTML uses HTML markup to reinforce the meaning of the content.

Semantic Elements

Semantic elements clearly describe their meaning to both the browser and the developer. Examples include:

  • <header>: Represents introductory content, typically a group of introductory or navigational aids.
  • <nav>: Contains navigation links for the document.
  • <article>: Represents a self-contained piece of content that could be distributed independently.
  • <section>: Represents a thematic grouping of content, typically with a heading.
  • <footer>: Represents the footer for its nearest sectioning content or sectioning root element.
  • <aside>: Represents content that is tangentially related to the content around it, often used for sidebars.
  • <figure>: Represents self-contained content, like illustrations, diagrams, or photos.
  • <figcaption>: Represents a caption or legend for a <figure> element.

Using semantic HTML improves accessibility and SEO, as search engines can better understand the content structure.

Multimedia

HTML5 introduced native support for audio and video, allowing for rich media experiences without the need for plugins.

Audio Example

To embed audio, use the <audio> tag:

    <audio controls>  
        <source src="audio.mp3" type="audio/mpeg">  
        Your browser does not support the audio tag.  
    </audio>

The controls attribute adds playback controls, and you can specify multiple sources for different formats.

Video Example

To embed video, use the <video> tag:

    <video width="320" height="240" controls>  
        <source src="movie.mp4" type="video/mp4">  
        Your browser does not support the video tag.  
    </video>

You can also use attributes like autoplay, loop, and muted to control playback behavior.

Canvas & SVG

HTML5 introduced the <canvas> element for drawing graphics on the fly via JavaScript.

Canvas Example

To create a canvas, use the <canvas> tag:

    <canvas id="myCanvas" width="200" height="100"></canvas>  
    <script>
    var canvas = document.getElementById("myCanvas");  // Get the canvas element
    var ctx = canvas.getContext("2d");  // Get the 2D drawing context
    ctx.fillStyle = "red";  // Set fill color
    ctx.fillRect(0, 0, 150, 75);  // Draw a filled rectangle
    </script>

The getContext("2d") method is used to obtain a drawing context for 2D rendering.

SVG Example

Scalable Vector Graphics (SVG) is another way to create graphics:

    <svg width="100" height="100">  
        <circle cx="50" cy="50" r="40" stroke="black" stroke-width="3" fill="red" />  
    </svg>

SVG graphics are resolution-independent and can be scaled without losing quality.

Geolocation

The Geolocation API allows the user to share their location with web applications.

Geolocation Example

To get the user's current position, use the following code:

    if (navigator.geolocation) {
        navigator.geolocation.getCurrentPosition(function(position) {
            alert("Latitude: " + position.coords.latitude + 
                  ", Longitude: " + position.coords.longitude);  // Display coordinates
        });
    } else {
        alert("Geolocation is not supported by this browser.");  // Fallback for unsupported browsers
    }

This code checks if geolocation is supported and retrieves the user's current latitude and longitude.

Drag & Drop

HTML5 provides a drag-and-drop API that allows users to drag elements and drop them in different locations.

Drag & Drop Example

Here’s a simple example of drag-and-drop functionality:

    <div id="drag1" draggable="true" ondragstart="drag(event)">Drag me!</div>  
    <div id="div1" ondrop="drop(event)" ondragover="allowDrop(event)">Drop here</div>  
    
    <script>
    function allowDrop(ev) {
        ev.preventDefault();  // Prevent default behavior to allow drop
    }
    function drag(ev) {
        ev.dataTransfer.setData("text", ev.target.id);  // Store the id of the dragged element
    }
    function drop(ev) {
        ev.preventDefault();  // Prevent default behavior
        var data = ev.dataTransfer.getData("text");  // Get the id of the dragged element
        ev.target.appendChild(document.getElementById(data));  // Append the dragged element to the drop target
    }
    </script>

This example demonstrates how to make an element draggable and how to handle the drop event.

Web Storage

Web Storage provides storage that is accessible within the user's browser, allowing for persistent data storage.

Local Storage Example

Local storage allows you to store data with no expiration date:

    localStorage.setItem("name", "John Doe");  // Store data
    var name = localStorage.getItem("name");  // Retrieve data
    alert(name);  // Display the stored name

Data stored in local storage persists even after the browser is closed and reopened.

Session Storage Example

Session storage is similar to local storage, but data is cleared when the page session ends:

    sessionStorage.setItem("sessionName", "Jane Doe");  // Store data for the session
    var sessionName = sessionStorage.getItem("sessionName");  // Retrieve session data
    alert(sessionName);  // Display the stored session name

Web Workers

Web Workers allow for background scripts to run in parallel to the main thread, improving performance for complex tasks.

Web Worker Example

To create a web worker, you need a separate JavaScript file:

    // worker.js
    self.onmessage = function(event) {
        var result = event.data * 2;  // Perform a calculation
        self.postMessage(result);  // Send the result back to the main thread
    }

In your main script, you can create and communicate with the worker:

    var worker = new Worker("worker.js");  // Create a new worker
    worker.onmessage = function(event) {
        console.log("Result from worker: " + event.data);  // Handle the result
    };
    worker.postMessage(10);  // Send data to the worker

Server-Sent Events (SSE)

SSE allows a server to push updates to the client, enabling real-time updates.

SSE Example

To use SSE, you need to set up a server endpoint that sends events:

    var eventSource = new EventSource("events.php");  // Create a new EventSource
    eventSource.onmessage = function(event) {
        console.log("New message: " + event.data);  // Handle incoming messages
    };

The server should send data in the following format:

    data: Your message here\n\n  // Format for sending data
